# What is Good Design?  
  
## Easy to maintain  
  
* Methods with specific tasks  
* Adaptable  
* Uses inheritance in a "good way  
    * Express is-a relationships  
    * No duplicated code  
* Small methods  
* Easy to read and understand  
    * Well chosen names  
    * Consistent naming  
        * eg CamelCase  
    * Class well documented and clear responsibilities  
* Encapsulation
